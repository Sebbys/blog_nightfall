### A. Git Definition
Git is a version control system that allows developers to track changes made to their codebase over time. It’s a distributed system, meaning every developer working on a project has a local copy of the entire project history, making collaboration easy both online and offline.

Git is also known as a distributed revision control system (VCS), meaning the Git database isn’t just stored in one location. Every person involved in coding the project will store a Git database, making it easier to manage the project both online and offline.

#### Basic Concepts
1. **Repository (Repo):** A Git repository is where all files and project history are stored.
2. **Commit:** A commit is a snapshot of the project at a specific point in time. When you make changes to the code, you commit those changes to the repository.
3. **Branch:** A branch is a separate line of development within the repository. You can create multiple branches to work on different features or fixes without affecting the main codebase.
4. **Merge:** When you’re ready to combine changes from one branch into another, you merge the two branches.
5. **Push:** When you push changes to a remote repository, you update the remote repository with your local changes.
6. **Pull:** When you pull changes from a remote repository, you update your local repository with changes from the remote repository.

#### How Git Works
- **Initialize a repository:** You create a new Git repository by running `git init` in your project directory.
- **Add files:** You add files to the repository using `git add <file>` or `git add .` to add all files in the directory.
- **Commit changes:** You commit changes to the repository using `git commit -m "commit message"`.
- **Create a branch:** You create a new branch using `git branch <branch-name>`.
- **Switch to branch:** You switch to a branch using `git checkout <branch-name>`.
- **Make changes:** You make changes to the code and commit them to the branch.
- **Merge branch:** You merge changes from one branch to another using `git merge <branch-name>`.
- **Push changes:** You push changes to the remote repository using `git push origin <branch-name>`.
- **Pull changes:** You pull changes from the remote repository using `git pull origin <branch-name>`.

### C. GitHub Definition
GitHub is a platform that allows developers to store and access code in a single place. It facilitates code management online, either individually or when collaborating with a team. GitHub allows you to create repositories, branches, commits, pull requests, and manage code online.

It also provides cloud services to store and manage Git repositories, allowing teams to collaborate on projects from different locations.

Configure your GitHub account’s name and email:

```bash

git config --global user.email contoh@gmail.com
```

Verify the configuration using:

```bash
Copy code

```

### 4. Pushing a Repository
Navigate to the folder you want to push, open Command Prompt (cmd), and use the following commands:

<p align="center">
<img src="https://res.cloudinary.com/dtzqdo3p0/image/upload/v1727448671/Screenshot_1559_kzisjv.png" alt="Create Repository"/>
</p>

Copy code

```bash

git init
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/yourusername/yourproject.git
git push -u origin main
```
